# 코딩에그 (Coding Egg) 종합 프로젝트 기획안

## 1. 프로젝트 정체성
- **서비스명**: 코딩에그 (Coding Egg)
- **핵심 철학**: 듀오링고(Duolingo)의 게이미피케이션(Gamification) 방식을 코딩 교육에 이식. 어른과 아이 모두가 즐겁게 코딩을 배우고, 자기 문제를 스스로 해결하는 빌더(Builder)로 탄생하도록 돕는다.
- **본질**: 브라우저에서 직접 코드를 실행하고 피드백을 받는 '순수 코딩 스킬 트랙'과 실무 자격증 '마스터 트랙'이 결합된 플랫폼.

## 2. 학습 로드맵 (거대 스킬 트리 시스템)
코딩에그는 순차적으로 해금되는 스킬 트리 구조를 가진다.
- **[Skill Track] (자유 오픈)**: 파이썬(Python) 기초부터 시작하여 데이터 분석, 시각화, AI까지 단계별로 학습.
- **[Master Track] (선행 과정 완료 시 해금)**: AICE Associate, 리눅스 마스터 등 실무 자격증 대비 트랙.

## 3. 핵심 학습 시스템
- **듀오링고형 경로**: 지그재그(Path) 형태의 학습 맵. 한 유닛은 15문제 이상의 문제로 구성됨.
- **이론+실무 결합**: 좌측 에디터에서 코딩 실습, 우측 패널에서 즉각적인 이론 학습 및 힌트 제공.
- **평가 방식**:
  - **중간/기말고사**: 유닛/섹션 마무리 시 보스전 배치.
  - **수료증**: 커리큘럼 완료 시 수료증 발급.
- **오답 시스템**: 틀린 문제는 레슨 마지막에 재등장하여 100% 숙달을 강제함.

## 4. 기술 스택
- **Frontend**: React/Next.js 기반, Tailwind CSS 사용.
- **Code Editor**: Monaco Editor (VS Code 경험 이식).
- **Runtime**: Pyodide (브라우저 내 파이썬 실행 및 시각화).
- **Database/Backend**: Supabase (데이터 관리).

## 5. AI 에이전트(Antigravity) 행동 수칙
- **원자적 개발**: 전체를 한 번에 만들지 않고, UI, 채점기, DB 등 각 조각을 분리하여 구현함.
- **플랜 모드(Plan First)**: 어떠한 코드 작성 전 반드시 `[Implementation Plan]`을 작성하여 인간 빌더의 승인을 받을 것.
- **지속적 문서화**: `AGENTS.md` 파일을 항상 최신 상태로 유지하여, 에이전트가 프로젝트 맥락을 잃지 않게 함.
- **가시성 확보**: 코드 수정 시 변경된 파일과 이유를 인간 빌더가 이해하기 쉽게 설명함.

## 6. 대화 맥락 요약
- 파이썬 트랙부터 순수하게 시작하여 듀오링고 방식의 학습 재미를 확보함.
- 이후 AI(머신러닝/딥러닝/LLM) 트랙으로 확장.
- 최종적으로 AICE 자격증 대비 과정을 마스터 트랙으로 배치함.
- 현재 '메인 스킬 트리 대시보드 UI'의 뼈대 구현을 위해 안티 그래비티 에이전트와 작업 중.
